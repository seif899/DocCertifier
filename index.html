Processus de certification
01
Hash local
Le fichier est haché avec keccak256 dans le navigateur. Aucune donnée ne quitte votre machine.
02
Signature MetaMask
MetaMask signe la transaction avec votre clé privée et vous demande confirmation.
03
Enregistrement
Le hash, l'adresse émettrice et l'horodatage sont inscrits de façon permanente sur la blockchain.
04
Vérification libre
N'importe qui peut vérifier n'importe quel document sans wallet. Lecture gratuite.

Fonctions principales exposées par le smart contract
// Certification — modifie l'état, coûte du gas
function certify (bytes32 docHash, string metadata) external

// Vérification — lecture seule, gratuite
function verify (bytes32 docHash) external view
→ issuer, timestamp, metadata, revoked

// Révocation — réservée à l'émetteur original
function revoke (bytes32 docHash) external

// Événements émis
event DocumentCertified (bytes32 indexed, address indexed, uint256, string)
event DocumentRevoked (bytes32 indexed, address indexed)
